# Приложение для управления рассылками

Это веб-приложение на Django для управления массовыми email-рассылками с поддержкой пользовательских аккаунтов, прав
доступа и статистики отправок.

## Основные возможности

- **Управление пользователями**: регистрация, авторизация, профили, подтверждение email
- **Создание сообщений**: тема и текст сообщения для рассылки
- **Управление получателями**: добавление, редактирование и удаление email-адресов получателей
- **Создание рассылок**: планирование отправки сообщений определенным получателям
- **Отправка рассылок**: ручной запуск процесса отправки
- **Мониторинг попыток**: отслеживание успешности отправок
- **Статистика**: аналитика по отправленным сообщениям

## Архитектура приложения

Приложение состоит из двух основных Django-приложений:

### `users`

Приложение для управления пользователями системы:

- **Модели**:
    - `CustomUser`: кастомная модель пользователя с полями email, телефон, аватар, страна, статус активации и токен
      активации
    - Поддержка аутентификации через email
    - Подтверждение регистрации по email
    - Группы пользователей: "Пользователь" и "Менеджер"

- **Функциональность**:
    - Регистрация с подтверждением по email
    - Авторизация и выход из системы
    - Просмотр и редактирование профиля
    - Удаление аккаунта
    - Смена пароля
    - Восстановление пароля по email
    - Администрирование пользователей (для менеджеров и суперпользователей)

### `sending`

Приложение для управления email-рассылками:

- **Модели**:
    - `Message`: сообщение для рассылки с темой и текстом
    - `MessageRecipient`: получатель с email, именем и комментарием
    - `Mailing`: рассылка с датой начала/окончания, статусом и связью с сообщением и получателями
    - `MailingAttempt`: попытка отправки с результатом, временем, временем окончания, ответом сервера и связью с
      рассылкой и получателем

- **Функциональность**:
    - Создание, редактирование и удаление сообщений
    - Управление списками получателей
    - Создание и планирование рассылок
    - Запуск отправки рассылок
    - Просмотр истории отправок и статусов
    - Блокировка/разблокировка сообщений, рассылок и получателей
    - Статистика по отправленным сообщениям

## Рабочий процесс

1. Пользователь регистрируется и подтверждает email
2. Создает сообщения для рассылки
3. Добавляет получателей в базу
4. Создает рассылку, выбирая сообщение и получателей
5. Запускает отправку рассылки
6. Просматривает результаты отправки и статистику

## Права доступа

Приложение использует систему разрешений Django:

- **Обычные пользователи**: имеют права на создание, просмотр, изменение и удаление своих сообщений, рассылок и
  получателей, а также на просмотр и изменение своего профиля
- **Менеджеры**: имеют дополнительные права на просмотр всех объектов и управление пользователями
- **Суперпользователи**: полный доступ ко всем функциям

## Технические особенности

- **Кэширование**: используется Redis для кэширования часто запрашиваемых данных (настроено в `settings.py`)
- **Логирование**: подробное логирование операций в файл `sending_app.log` и в консоль
- **Валидация email**: использование библиотеки `email-validator` для проверки корректности email-адресов
- **Отправка email**: через SMTP с обработкой ошибок и записью результатов
- **Интерфейс**: шаблоны Django с использованием встроенных механизмов

## Запуск приложения

1. Установите зависимости: `poetry install`
2. Настройте базу данных PostgreSQL
3. Настройте переменные окружения в `.env` файле
4. Выполните миграции: `python manage.py migrate`
5. Создайте суперпользователя: `python manage.py createsuperuser`
6. Альтернатива пункту 5 создать пользователей кастомной командой: `python manage.py cau`
7. Запустите сервер: `python manage.py runserver`

## URL-маршруты

### Пользователи (`/users/`)

- `/register/` - регистрация
- `/login/` - вход в систему
- `/logout/` - выход из системы
- `/profile/<int:pk>/` - просмотр профиля
- `/profile/update/<int:pk>/` - редактирование профиля
- `/profile/delete/<int:pk>/` - удаление профиля
- `/activate/<int:pk>/<str:token>/` - активация аккаунта
- `/list/` - список пользователей (для менеджеров)

### Рассылки (`/`)

- `/` - главная страница
- `/contact/` - страница контактов
- `/messages/` - список сообщений
- `/message/create/` - создание сообщения
- `/mailing/list/` - список рассылок
- `/mailing/create/` - создание рассылки
- `/recipients/` - список получателей
- `/recipient/create/` - создание получателя
- `/attempts/<int:pk>/` - попытки отправки для рассылки
- `/send_mailing/<int:pk>/` - ручной запуск отправки рассылки
- `/statistics/` - статистика по рассылкам

## Безопасность

- Все пользователи регистрируются с подтверждением email
- Пароли хранятся в хешированном виде
- Используется защита от CSRF
- Поддержка HTTPS через настройки email (TLS/SSL)
- Ограничение доступа к объектам на основе владельца и прав

## Логирование

Приложение использует систему логирования с двумя обработчиками:

- Консольный вывод для отладки
- Запись в файл `sending_app.log` для долгосрочного хранения

Уровни логирования настроены отдельно для разных компонентов:

- `django`: INFO
- `sending`: DEBUG
- `users`: DEBUG

## Кастомные команды

Приложение содержит следующие кастомные Django-команды:

### `cau` (create_admin_user)

- **Назначение**: Создает трех тестовых пользователей с разными ролями и настраивает группы с соответствующими правами
  доступа
- **Пользователи**: admin, manager, user
- **Пароль**: qwer1234
- **Группы**: Администратор, Менеджер, Пользователь
- **Расположение**: `users/management/commands/cau.py`
- **Использование**: `python manage.py cau`

### `send`

- **Назначение**: Выполняет отправку активных рассылок в соответствии с их расписанием
- **Логика**: Проверяет условия отправки (временные рамки, активность рассылки)
- **Расположение**: `sending/management/commands/send.py`
- **Использование**: `python manage.py send`
- **Типичное использование**: В cron или планировщике задач для периодической отправки рассылок
